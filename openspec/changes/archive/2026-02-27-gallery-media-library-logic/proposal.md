# 相册模块逻辑变更

## Why

用户希望相册与系统相册解耦：应用拥有独立的媒体库，拍照仅写入应用内，由用户主动选择将照片保存到系统相册；同时优化相册浏览体验，默认以列表展示、滚动时懒加载与渐现、点击进入全屏大图并支持左右滑动浏览。

## What Changes

- **应用独立媒体库**：相册不再直接读取系统相册；媒体数据来源于应用自有存储（如应用文档目录或仅限本应用的相册），拍照完成后只写入该媒体库。
- **保存到系统相册**：通过长按单张或批量选择多张后，提供「下载到系统相册」能力，将选中的照片写入设备系统相册。
- **从应用媒体库删除**：支持单张删除（长按菜单）与批量选择后删除；仅从应用媒体库移除（删文件与元数据），不涉及系统相册；不做左滑删除。大图页删除当前张后显示下一张，若无下一张则退回列表。
- **默认列表与体验**：打开媒体库时默认显示列表视图；列表支持上下滑动，滑动过程中对尚未进入视口的图片做懒加载，并为新加载的图片增加渐现（fade-in）效果。
- **全屏大图与左右滑动**：点击列表中的某张图进入全屏大图；大图视图支持左右滑动切换查看上一张/下一张。

## Capabilities

### New / Modified Capabilities

- **应用媒体库**：媒体库数据源改为应用内存储（文件系统或专用相册），不依赖系统相册读取；拍照后仅写入应用媒体库。
- **保存到系统相册**：长按单张或进入批量选择模式勾选多张后，提供「下载到系统相册」操作；仅在此操作时请求系统相册权限并写入。
- **应用媒体库删除**：长按单张可弹出菜单选择「删除」；批量选择模式下提供「删除」按钮；删除仅从应用媒体库移除，需二次确认；大图页删除当前张后自动显示下一张，无下一张时退回列表。不实现左滑删除。
- **相册默认列表**：相册页默认进入列表视图（不再默认进入最新一张大图）；列表为可上下滑动的列表布局。
- **列表懒加载与渐现**：列表使用懒加载（如 FlatList / 虚拟列表），仅渲染可见及临近项；图片加载完成后使用渐现动画提升体验。
- **全屏大图与左右滑动**：点击列表项进入全屏大图；大图页支持左右滑动切换相邻图片（如使用 FlatList horizontal 或 ViewPager 形式）。

## Impact

- **影响范围**：`features/gallery` 的媒体数据源与读写逻辑（mediaLibrary、Context、删除接口）、相册页默认视图与路由逻辑（app/gallery）、列表组件（懒加载 + 渐现、长按菜单与批量删除）、大图组件（全屏 + 左右滑动、删除后显示下一张）；CameraBottomBar 相册入口仍显示应用媒体库首张缩略图。后续优化：保存到系统相册的成功/失败与批量进度反馈；GalleryImageView 标注为遗留。
- **依赖**：expo-file-system 或继续使用 expo-media-library 但仅写入/读取「本应用相册」；系统相册写入仍依赖 expo-media-library 权限与 createAssetAsync。
