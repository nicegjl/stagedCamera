# 优化媒体库逻辑（相册 UX）

## Why

在现有应用媒体库、列表/大图、批量选择与保存到系统相册能力基础上，优化相册页的布局与交互：多选工具栏更紧凑（右上角 + 图标）、大图页可显式唤起操作菜单、列表与大图的返回入口区分清晰、大图支持拖动平移并带缓动以贴近系统相机体验。

## What Changes

- **多选工具栏位置与形态**：多选图片时，原底部/独立一栏的「下载到系统相册」「删除」移至**右上角**（原「选择」按钮所在区域）；「下载」与「删除」改为**图标**展示，保留进度与二次确认逻辑。
- **大图操作入口**：大图展示时，除长按外，提供显式入口以唤起「删除/下载到系统相册」的同一套操作菜单（如右上角更多/菜单图标，或保留长按并确保文案一致）。
- **返回入口区分**：**列表展示时**左上角为「返回 icon + 相机」（点击返回可回到上一页，相机可跳转相机 tab）；**大图浏览时**左上角保持「返回」回到列表。通过布局区分两种返回行为。
- **大图拖动与缓动**：全屏大图在缩放或默认状态下，支持**手指拖动**平移查看图片；位移使用**缓动动画**（easing），整体体验类似 iOS 系统相册/相机的拖动与回弹感。

## Capabilities

### Modified Capabilities

- **相册列表多选工具栏**：多选时工具栏位于顶部右侧（原「选择」位），下载与删除为图标按钮；保存中仍显示进度（如 N/M 张）。
- **大图操作菜单**：大图页可通过显式入口（如右上角菜单）或长按唤起「保存到系统相册」「删除」操作菜单，与列表/批量行为一致。
- **相册页头部导航**：列表模式左上角 = 返回 icon + 相机 icon；大图模式左上角 = 返回（回列表）。
- **大图平移与缓动**：大图支持拖动手势平移；位移与松手回弹使用缓动曲线（如 iOS 风格），提升手感。

## Impact

- **影响范围**：`app/gallery.tsx`（头部布局、多选工具栏位置与图标化）、`features/gallery/components/GalleryDetailView.tsx`（大图操作入口、拖动手势与缓动）；可能复用或扩展 `@expo/vector-icons` 图标。
- **依赖**：现有 react-native-gesture-handler、react-native-reanimated；无需新增依赖。
