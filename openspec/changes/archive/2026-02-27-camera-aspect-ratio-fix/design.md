# 修复画幅调整的逻辑 - 设计

## Context

- 画幅类型：`AspectRatio = '4:3' | '16:9' | '1:1'`（已移除 full），由 CameraContext 管理，CameraTopBar 切换。
- 应用为竖屏锁定（`orientation: portrait`），故 4:3、16:9 在取景与成片上按 **3:4、9:16** 实现。
- 目标：预览区域与成片均与所选画幅一致；仅三档固定比例，无「全屏/原图」选项。

## Goals / Non-Goals

**Goals:**

- 随画幅调整，相机可视区域（取景框）的宽高比与 3:4、9:16、1:1 一致（竖屏）；若与屏幕比不同，用黑边或裁剪预览实现。
- 拍摄完成的照片宽高比与当前画幅一致（中心裁剪，竖屏下 3:4、9:16、1:1）。

**Non-Goals:**

- 不恢复 full 档位；不改变 CameraContext/CameraTopBar 的交互方式（仍为循环切换三档）。

## Decisions

### 1. 画幅选项与竖屏比例

- **画幅仅保留三档**：4:3、16:9、1:1；移除 full。默认 4:3。
- **竖屏下比例**：应用竖屏锁定，选项「4:3」对应实际比例 **3:4**（高/宽），「16:9」对应 **9:16**，「1:1」仍为 1:1。取景框与成片均按上述比例计算/裁剪。

### 2. 预览区域随画幅调整

- 在可用区域内居中一个「固定比例框」，比例值为竖屏下的有效比例（3/4、9/16、1）。
- 以「短边贴满、长边留黑」得到该比例的最大内接矩形，CameraView 限制在该矩形内（取景框容器 + `overflow: 'hidden'`）。
- 取景框尺寸：用 `onLayout` 得到预览容器宽高，按有效比例算出取景框宽高并居中。

### 3. 成片满足画幅

- 采用「拍原图 + 中心裁剪」：`takePictureAsync` 得到原图后，用 `expo-image-manipulator` 按当前画幅（竖屏下 3:4、9:16、1:1）从中心裁剪，再 `savePhoto`。无 full，故每次成片均为固定比例。

### 4. 与模板/九宫格叠加层的关系

- GridOverlay、TemplateOverlay 覆盖在同一取景框区域上（与预览同层级或同父级），避免错位。

## Risks / Trade-offs

- 预览与成片需使用同一套「有效比例」逻辑（竖屏 3/4、9/16、1），保证所见即所得。
- 若将来支持横屏，需按朝向切换为 4/3、16/9 或 3/4、9/16。
