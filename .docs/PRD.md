# stagedCamera 产品需求文档（PRD）

**版本**：v0.1  
**更新日期**：2025-02-25  
**产品定位**：移动端相机应用，主打「人物拍照模版」引导摆姿，同时提供完整常规拍摄能力。

---

## 一、项目概述

### 1.1 产品简介

stagedCamera 是一款面向移动端（iOS/Android）的相机应用，核心价值在于：

1. **常规相机能力**：提供与系统相机相当的拍照、录像及基础调节能力，满足日常拍摄。
2. **人物拍照模版**：通过半透明人物线框图叠加在取景画面上，引导用户「照着姿势摆拍」，降低构图与摆姿门槛，适用于旅行、打卡、合影、节日等场景。

### 1.2 目标用户

- 需要「照着摆姿势」的普通用户（旅行打卡、景点合影、节日照等）。
- 对成片构图和姿势有要求、但不想记忆复杂姿势的用户。
- 需要快速统一多人合影姿势的组织者或摄影师。

### 1.3 核心场景

- 在景点/地标前按模版姿势单人/多人拍照。
- 节日、活动、场地（婚礼、毕业、展会）下的标准化摆姿。
- 日常自拍、合影时快速获得「专业感」构图与姿势。

## 三、常规相机功能（详细）

以下按「必备 / 常用 / 增强」划分，便于排期与 MVP 裁剪。

### 3.1 拍摄模式与基础能力

| 功能点       | 描述 | 优先级 |
|--------------|------|--------|
| **拍照**     | 单张拍摄，按下快门完成一次拍照并保存到系统相册（或应用内相册）。 | P0 |
| **录像**     | 开始/停止录制视频，界面显示录制时长与状态；支持短时长限制或提示（如 1 分钟/5 分钟）。 | P0 |
| **前后摄像头切换** | 在前置与后置摄像头之间切换，按钮位置符合常见习惯（取景区域旁或顶部栏）。 | P0 |
| **闪光灯**   | 模式：关闭 / 开启 / 自动；仅在拍照模式生效（录像时可保留「常亮补光」作为增强）。 | P0 |
| **相册入口** | 从相机界面进入「最近照片/视频」或系统相册，可查看本应用拍摄内容及从相册选图（若产品需要）。 | P0 |

### 3.2 取景与构图

| 功能点       | 描述 | 优先级 |
|--------------|------|--------|
| **变焦**     | 支持手势（双指捏合）变焦；可选固定档位（如 0.5x / 1x / 2x）或平滑变焦；与设备能力一致（广角/超广/长焦若存在）。 | P0 |
| **对焦**     | 点击取景区域进行单点对焦；默认连续自动对焦（AF）；若有能力可考虑「对焦锁定」。 | P0 |
| **曝光调节** | 对焦后或独立滑动条调节曝光补偿（EV），实时预览反馈。 | P1 |
| **构图辅助线** | 九宫格网格叠加在取景画面上，可开关；可选对角线或其它辅助线。 | P1 |
| **画幅比例**  | 可选 4:3、16:9、1:1、全屏等，切换后取景与成片比例一致。 | P1 |

### 3.3 拍摄辅助

| 功能点       | 描述 | 优先级 |
|--------------|------|--------|
| **倒计时拍摄** | 按下快门后延迟 3 秒或 10 秒再拍摄，便于自拍/合影；界面有倒计时提示。 | P1 |
| **水平仪/参考线** | 可选的水平仪或建筑参考线，辅助横平竖直构图（建筑/风景场景）。 | P2 |
| **HDR**      | 自动或手动 HDR 拍照，提升大光比场景细节（依赖系统/硬件能力）。 | P2 |
| **连拍**     | 长按快门进行连拍，并支持「从连拍中选一张或一组保存」。 | P2 |

### 3.4 录像相关（增强）

| 功能点       | 描述 | 优先级 |
|--------------|------|--------|
| **分辨率/帧率** | 设置录制分辨率（如 1080p/4K）与帧率（如 30fps/60fps），依赖平台能力。 | P2 |
| **录像防抖** | 若系统提供电子/光学防抖，在设置中可开关或自动。 | P2 |

### 3.5 系统与设置

| 功能点       | 描述 | 优先级 |
|--------------|------|--------|
| **权限与说明** | 相机、麦克风、相册读写权限申请与说明文案；无权限时引导用户去系统设置开启。 | P0 |
| **基础设置** | 如：拍照分辨率、默认画幅、网格开关、快门声音开关、保存位置（相册/仅应用内）等。 | P1 |
| **存储与性能** | 照片/视频写入相册或应用沙盒；大文件与长时间录像的存储与性能考虑。 | P0 |

### 3.6 常规相机功能小结（清单）

- **P0（MVP 必备）**：拍照、录像、前后摄切换、闪光灯、相册入口、变焦、对焦、权限与存储。
- **P1（首版尽量有）**：曝光调节、构图网格、画幅比例、倒计时、基础设置。
- **P2（后续迭代）**：水平仪、HDR、连拍、录像分辨率/帧率、防抖等。

## 四、人物拍照模版（详细）

### 4.1 功能概述

- **形态**：在相机取景画面上叠加**半透明人物线框图**（可多个人形），用于描述「期望的姿势与位置」。
- **流程**：用户选择模版 → 取景器上显示对应线框图 → 用户（及同伴）按线框摆姿 → 对齐后拍摄。
- **目标**：降低「不知道摆什么姿势、站哪里」的门槛，提升打卡、合影、节日照的成片率和统一感。

### 4.2 核心能力

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| **模版库** | 内置一组人物姿势模版，每个模版包含：一张或多张半透明人物线框图、推荐画幅、简要说明。 | P0 |
| **模版选择** | 从模版库中浏览并选择当前要用的模版；支持列表/网格展示与基础分类入口。 | P0 |
| **取景叠加** | 在实时取景画面上叠加当前模版的线框图，位置与缩放可适配屏幕比例；线框半透明、不遮挡过多画面。 | P0 |
| **线框可调** | 用户可调节线框的**透明度**（如 30%～90%）和**大小/位置**（整体缩放、上下平移），以便适配不同身高与取景距离。 | P1 |
| **拍摄与保存** | 在模版叠加状态下直接拍照；成片**不包含线框**（仅保存真实画面）；可选择性保存「带线框的预览图」供分享或对比。 | P0 |
| **模版与常规模式切换** | 在「常规相机」与「人物模版」模式间一键切换，不打断已有拍摄流程。 | P0 |

### 4.3 模版扩展性设计（重要）

模版需支持多维度扩展，便于后续增加场景、人数、风格等，而不必推翻已有架构。

#### 4.3.1 扩展维度

| 维度 | 说明 | 示例 |
|------|------|------|
| **场景/用途** | 按使用场景分类，便于运营与推荐。 | 背景（自然/城市）、建筑、场地（婚礼/毕业/展会）、节日（春节/圣诞/万圣节） |
| **人数** | 模版支持的人体数量，影响线框数量与布局。 | 单人、双人、多人（3～5 人、群像） |
| **姿势类型** | 站姿、坐姿、蹲姿、跳跃、互动（牵手/举杯）等。 | 站姿、坐姿、互动、创意/搞怪 |
| **画幅与构图** | 横图/竖图、全身/半身、居中/三分等。 | 竖版全身、横版三人、方形居中 |
| **风格/主题** | 文艺、运动、正式、趣味等，用于标签与筛选。 | 文艺、运动、正式、节日 |

#### 4.3.2 模版数据结构建议（便于扩展）

```text
模板 (Template)
├── id: 唯一标识
├── name: 展示名称
├── description: 简短说明（可选）
├── categoryIds: [场景, 人数, 姿势类型, ...]  // 多维度分类
├── tags: [关键词]                             // 搜索与推荐
├── figures: [                                 // 人物线框列表（支持多人）
│   {
│     image: 线框资源 URL/本地路径,
│     defaultScale: 默认缩放,
│     defaultOffset: { x, y },                 // 默认在画布上的偏移
│     roleHint: "左/中/右" 或 "人物1/人物2"    // 可选，用于多人的左右顺序
│   }
│ ]
├── aspectRatio: "4:3" | "16:9" | "1:1" | "full"  // 推荐画幅
├── previewImage: 模版预览图（列表缩略图）
└── sortOrder: 排序权重
```

- **figures** 为数组：单人一个元素，双人/多人多个元素，便于后续加「三人合影」「家庭照」等。
- **categoryIds / tags** 用于筛选、搜索与「按场景/节日/人数」扩展。
- **aspectRatio** 与 **defaultScale/defaultOffset** 保证在不同屏幕与画幅下有一致体验，并可做简单自动适配。

#### 4.3.3 内容扩展方式

- **内置模版**：随应用发布一批精选模版，覆盖单人、双人、常见场景与节日。
- **运营扩展**：通过配置或远程拉取模版列表 + 资源包，新增「节日专题」「地标合作」等而不发版。
- **未来可考虑**：用户自定义模版（上传线框图或从照片生成轮廓）、UGC 模版分享（需合规与审核）。

### 4.4 人物模版交互要点

- **进入模版模式**：首页或相机顶部有「模版」入口，进入后先选模版再进入取景；或先取景再选模版叠加。
- **选模版**：列表/网格展示，支持按「场景 / 人数 / 节日」等筛选项；点击模版即应用并回到取景（若已设计为「先选再拍」）。
- **取景页**：中央为实时画面，其上叠加当前模版线框；提供「透明度」「大小/位置」调节（滑块或双指）；快门、前后摄、闪光等与常规模式一致。
- **引导文案**：首次使用或进入某类模版时，可简短提示「请按线框摆好姿势后拍摄」。

### 4.5 人物模版非功能考虑

- **性能**：叠加层轻量渲染（静态或简单动画），避免掉帧。
- **资源**：线框图为矢量或高质量位图，控制包体与下载量；远程模版可做懒加载与缓存。
- **无障碍**：线框与背景对比度可调（透明度），重要操作支持读屏与明确焦点。

## 五、非功能需求（概要）

| 类型 | 要求 |
|------|------|
| **平台** | 支持 iOS、Android；技术栈与现有项目（Expo/React Native）一致。 |
| **性能** | 取景与拍照响应流畅，录像不因应用层逻辑明显卡顿。 |
| **兼容** | 适配主流分辨率与安全区；不同机型摄像头能力差异可降级（如无超广则无 0.5x）。 |
| **权限与隐私** | 仅申请相机、麦克风、相册等必要权限；本地优先，若涉及云端模版需说明数据与隐私策略。 |
| **稳定性** | 异常（无权限、存储满、摄像头占用）有明确提示与恢复路径。 |

## 六、版本与优先级建议

| 阶段 | 范围 | 说明 |
|------|------|------|
| **MVP** | 常规相机 P0 + 人物模版 P0（模版库、选择、叠加、无线框成片、模式切换） | 先跑通「选模版 → 叠加 → 摆姿 → 拍照」完整闭环。 |
| **V1.0** | 常规 P1（曝光、网格、画幅、倒计时、设置）+ 模版 P1（线框透明度/大小调节） | 提升常规拍摄体验与模版可调性。 |
| **V1.x** | 模版扩展（更多分类、人数、节日/场景）+ 常规 P2（HDR、连拍等） | 丰富模版内容与专业功能。 |

## 七、附录

### 7.1 术语

- **线框图**：本 PRD 中指用于姿势引导的半透明人物轮廓图，非 UI 线框图。
- **画幅**：取景与成片的宽高比，如 4:3、16:9、1:1。
- **模版**：一组预定义的姿势线框及元数据，可包含单人或多人。
